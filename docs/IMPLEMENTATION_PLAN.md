# Implementation Plan

## Current project state (audit)
- Working directory was **empty** at start. No existing Xcode project, no git history.
- This is therefore a greenfield build, not a refactor.
- No prior functionality to preserve.

## Tooling baseline
| Tool | Version (verified) |
|---|---|
| Xcode | 26.4.1 |
| Swift | 5.10 |
| iOS SDK | 26 |
| iOS minimum deployment | 17.0 (set in `project.yml`) |
| Project generator | xcodegen 2.45 (`brew install xcodegen`) |
| Simulator used | iPhone 17 (`xcrun simctl`) |

## Architecture chosen
- SwiftUI + lightweight MVVM (`@Observable` viewmodels under iOS 17).
- Persistence: `UserDefaults` (config + onboarding flag), `SwiftData` (quiz history), bundled JSON (demo ranges).
- No third-party dependencies.

## Source layout
See `CLAUDE.md` § Architecture for the full tree. Key directories:
- `Cutoff/Theme/` — `AppColors`, `AppTypography`, `AppSpacing`, `AppRadius`, `AppMotion`, `AppGlass`, `AppTheme`

## Verification (acceptance criteria)
See the spec in the user's request; condensed here:
- App builds successfully.
- Tabs work; user can complete onboarding.
- Default profile renders **125 BB** for `25_000 / 200`.
- Preflop quiz answers produce immediate feedback with best answer + short reason.
- 13×13 range grid renders, is filterable, and cells label demo provenance.
- Push/Fold trainer is usable.
- Review tab shows leak cards.
- All bundled ranges labeled demo / not solver-verified.
- Reduce Transparency: glass collapses to solid surfaces.
- Reduce Motion: animations are short-circuited.
- Dynamic Type up to `accessibility3` does not break layout on primary screens.
- All six test files pass.
